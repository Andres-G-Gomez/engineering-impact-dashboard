Review Influence Score

Measuring engineering leverage through high-quality code review — not lines written.

The Review Influence Score captures the leverage an engineer creates when they review code. A pull request merged after a substantive review — especially one in a critical subsystem that prompted the author to change their code — represents compounded impact: the reviewer's knowledge multiplied across every user who benefits from that fix or feature.

①
PR Complexity
Scores 1–5 based on files changed and total line delta. Bigger, riskier PRs earn higher multipliers.
②
Subsystem Criticality
Core paths like /hogql, /plugin-server, /posthog/api carry 2–3× weight multipliers.
③
Review Depth
Reviews >30 words or "Request Changes" decisions signal substantive engagement, not rubber-stamping.
④
Reviewer Centrality
Engineers who review across many authors and subsystems act as knowledge bridges for the whole team.
⑤
Acceptance Weight
A 1.5× multiplier when the author commits new code after the review — proof the feedback had a real effect.
